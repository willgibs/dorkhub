- 2026-07-21 · M2.5 motion retrofit judgment calls: Dialog/DialogContent gained an `animated` prop (default true) so CommandDialog can pass `animated={false}` — the ⌘K palette shares Dialog's markup but the high-frequency-action rule bans its animation, verified in-browser that the command surface renders with zero animate/duration classes while every other Dialog consumer keeps them. AvatarStack's hover-return overshoot uses an explicitly-motion.md-sanctioned custom cubic-bezier (the one exception to the two named easings). "Adapted from transitions.dev" note on /design/motion scoped to the 6 overlay primitives only, not the whole page, since the rest (StatButton burst, AvatarStack overshoot, etc.) are original.
- 2026-07-21 · ui/collapsible grid-rows retrofit: needs `forceMount` on CollapsibleContent (else Radix unmounts on close before the CSS transition can play) and the animating grid element's className must stay fixed — never merged with a consumer's own className, since a bare `flex` there silently overrides `grid` and kills `grid-template-rows` entirely (any consumer spacing must live on an inner wrapper instead). Shipped broken on the first pass; caught via live browser DOM inspection (computed `display`/`grid-template-rows`), not by `pnpm verify`/`build` (both are blind to this — it's a runtime CSS-cascade bug, not a type or lint error).
- 2026-07-21 · `ui/button.tsx`/`ui/tabs.tsx` `transition-all` (motion.md violation) fixed same-day with scoped transition-[…] lists; review-animations gate then drove: global `@custom-variant hover` gating (touch-safe hovers everywhere), stat-burst onto `--motion-slow`, staggered number pop-in, collapsible cost-comment correction. Accepted simplification: hover color transitions reuse ease-quiet (two-curve system) rather than adding a third `ease` curve.
