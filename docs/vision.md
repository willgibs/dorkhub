# dorkhub — Product Vision (board-approved 2026-07-22)

dorkhub is where developers **find tools they weren't looking for**. GitHub is
where you go when you know the name; dorkhub is the neighborhood you wander.

**Who it serves:** curators first (the silent majority — discover, save,
follow, organize), creators second (showcase, claim). Governance: Will = board
(approves vision, pricing, launch go, new spend, consent escalations); the
orchestrator agent = CEO (executes everything else to launch).

**Flywheel:** broad ingestion → 2-admin approval (Will + CEO only) fills the
gallery → curation traffic → creators list and claim their pages.

**Principles:**
1. Discovery over lookup — search exists (⌘K utility) but is demoted; the
   product's weight is on browse/tags/serendipity/recommendation surfaces.
2. Stars import is the activation moment ("bring your stars"). GitHub Lists
   have no API — excluded until that changes; no scraping.
3. Quality is the cap — human approval rate IS the volume throttle. Nothing
   auto-publishes. Import demand ranks the queue.
4. Unclaimed honesty + consent — badge, claim flow, owner/repo blocklist
   honored by every ingestion path, delete-and-block on request.
5. Cost discipline — $0/mo pre-launch; Vercel Pro + Spend Mgmt at launch;
   Supabase Pro only when forced (~1k READMEs); board notified pre-spend.

**Monetization:** featured slots in discovery surfaces only, clearly labeled.
Pricing = board gate before P4 ships it.

**Launch bar:** pixel-perfect full product. Content bar ~150–300 approved
projects across major tags. "Slow monetizable growth over 15 minutes of fame."

**Road:** P1 gallery engine (ingestion + stars import + claim) → P2 discovery
mechanics + search reframe → P3 rich pages + design-QA sweep → P4 featured
slots + hardening + dorkhub.com launch. Details: docs/plans/p*-*.md.
