## Locked decisions (do not re-litigate)

1. **Auto-approved encoding = `status='approved', decided_by NULL`** (humans
   always stamp decided_by; supersede trigger writes 'superseded', never
   'approved'). Retro queue = approved ∧ decided_by NULL ∧ stars <
   AUTO_APPROVE_MIN_STARS (env, default 20). "looks good" = stamp
   decided_by/decided_at. Raising the threshold retroactively surfaces items
   (accepted; bounded by LIMIT).
2. **No pre-publish content gate**: bare candidates publish immediately —
   og-image + name + stats IS the instant floor; AI tagline/tags follow
   within ~15 min. Pass 1 prefers content-having candidates; pass 2
   prioritizes published-bare.
3. **Supersede-trigger invariant**: our own project INSERT flips the pending
   candidate to 'superseded'; the follow-up status write MUST be
   unconditional (github_repo_id key only). Adding .eq('status','pending')
   would strand every materialized row as superseded.
4. **Profile-insert 23505 recovery**: concurrent materializers can race a
   new owner's profile creation — on 23505 re-select by github_id; only
   username_taken if that's empty.
5. **inlineEnrich opt-in**: human approve passes true (banner echo);
   pipeline + import phase-2 false (no unpaced AI inside materialization).
   ai_* precedence (human/GitHub data wins) stays in core unconditionally.
6. **stopKind machine field** ('rate_limited'|'config'|'provider_error'|
   null) drives auto-resume; stopReason stays display-only.
7. **Pacing**: ENRICH_PACE_MS=4500, ≤8 enrich calls/invocation, 50s soft
   deadline before every item (maxDuration 60). Actions cron
   `4,19,34,49 * * * *`; Vercel fallback `7 9 * * *`. ≈768 scheduled
   calls/day < ~1k Gemini free budget; GitHub ≈120 core calls/hr.
   At launch (Pro): delete pipeline.yml, Vercel cron goes `*/15`.
8. **Project enrichment is fill-only**: tagline when NULL, tags when '{}'
   (`.filter('topics','eq','{}')` — the `.eq(col, [])` gotcha); never
   overwrite; projects.enriched_at = provenance/no-retry (0008, service-only,
   rls_checks asserts). syncProject never touches these columns.
9. **Gallery default = trending**: `/` + `/home` sort=trending; 'newest'
   secondary chip; NEW /newest path (recent sort); /trending 308 → `/`;
   hrefs centralized in src/lib/feed/hrefs.ts. Comprehensive
   search/sort/filter = P3 note, not built now.
10. importDoneQueued + sortRecent copy keys are RETIRED once Wave 2C/1C land
    (kept green mid-wave); orchestrator deletes at Wave 3 sweep.
