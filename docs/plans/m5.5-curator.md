## Locked design decisions

1. **No PostgREST `.or()` in search** — its filter grammar treats `,`/`()` as
   syntax (injection surface beyond SQL wildcards). Paired `.ilike()` queries
   per field (6 total, one Promise.all), merged/ranked/deduped in pure code.
2. **Palette hand-composes** Dialog(animated={false}) + Command
   shouldFilter={false} + Input/List/Group/Item/Empty. Do NOT use
   CommandDialog (no shouldFilter passthrough; ui/ primitives are never
   edited). Results are server-filtered; cmdk's fuzzy filter must be OFF.
3. **Header stays server-rendered**; palette mounts ONCE in root layout;
   trigger↔dialog share a module-level external store consumed via
   useSyncExternalStore (precedent: site-header-auth module cache). No
   context providers, no DOM CustomEvents.
4. **Mobile header = pure-CSS two-row wrap** (flex-wrap + order-10 on nav +
   search trigger below sm). Logo + CTA/avatar line is UNTOUCHED (proven
   shipped layout). No Sheet/hamburger — out of scope.
5. Saved/following links show for EVERYONE on feed routes (trailing cluster);
   signed-out clicks ride the proxy's existing signin redirect (funnel).
   Feed pages stay ISR — zero cookie reads added anywhere.
6. Trigram indexes partial `where status='published'` on projects; opclass
   schema-qualified `extensions.gin_trgm_ops`; citext username indexed via
   `(username::text)` expression — EXPLAIN post-apply, drop if planner
   ignores it (profiles is small). No index on tiny `tags` table.
7. Copy: reuse savedTitle/followingTitle/browseCta/tagsTitle for links; new
   keys only for search (searchPlaceholder/searchEmpty/searchGroup*).
